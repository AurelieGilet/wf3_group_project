Projet de groupe WF3

Description du projet

Nous sommes donc parties sur l’idée d’un site permettant l'échange et la location de jeux de société et de jeux vidéo.
Il permettrait de mettre en contact, via l’interface du site, des loueurs et emprunteurs en fonction de leur département.

Il nous faut trouver une idée de nom.

Rôles dans l'equipe :

Véronique : front

Céline S. : back

Céline T. : plutôt back mais peut soutenir le front

Aurélie : plutôt back mais peut soutenir le front

Base de données

Bdd : "trouver un nom de bdd"

Table jeux_video :

“idJeuxVideo” INT(11) AI PK
"idPossesseur" INT(11) SK (secondary key liée à PK table membres)
“titre” VAR(255)
“support” ENUM (pc, ps3, ps4, switch, wii, wii-u, …)
“public” ENUM (pegi3, pegi7, pegi12, pegi16, pegi18)
“genre” ENUM (action, aventure, rpg, simulation, strategie, sport, reflexion, contemplatif)
“description” TEXT

Table jeux_de_societe :

“idJeuxSociete” INT(11) AI PK
"idPossesseur" INT(11) SK (secondary key liée à PK table membres)
“titre” VAR(255)
“public” ENUM (+6/7ans, +9/10ans, +12/14ans)
“nombreJoueurs” ENUM (1, 1-2, 2-4, 4-6, 6-8)
“genre” ENUM (adresse, memoire, cartes, connaissance, des, lettres, logique, strategie, plateaux, cooperation)
“description” TEXT

Table membres :

“idMembre” INT(11) AI PK
“pseudo” VAR(255)
“email” VAR(255)
“nom” VAR(255)
“prenom” VAR(255)
“adresse” VAR(255)
“code_postal” INT(5)
“ville” VAR(255)
“departement” INT(2)

Table emprunts_JeuxVideo :

“idEmprunt” INT(11) AI PK
“idMembreLoueur” INT(11) SK (secondary key liée à table membres)
“idMembreEmprunteur” INT(11) SK (secondary key liée à table membres)
“idJeux” INT(11) SK (secondary key liée à table jeux_video)

Table emprunts_jeuxSociete :

“idEmprunt” INT(11) AI PK
“idMembreLoueur” INT(11) SK (secondary key liée à table membres)
“idMembreEmprunteur” INT(11) SK (secondary key liée à table membres)
“idJeux” INT(11) SK (secondary key liée à table jeux_societe)

Interface du site

Page d'accueil :

  • liste des jeux
  • champs de recherche
  • système affichage par catégories (jeux video et jeux de société) et par sous catégories (pour chacune des catégories)

Page des jeux :

  • Affichage détaillé du jeux sélectionné
  • Liens vers option de location

Page emprunt :

  • location du jeux
  • réservation (système de file d’attente, pouvoir annuler sa réservation)
  • location pour 1 mois pour les jeux de plateau
  • location 2 mois pour les jeux vidéo
  • délai de 48h pour retirer le jeu

Page d’inscription membre :

  • formulaire d’inscription

Page espace membre :

  • profil
  • liste des empruntés (passés / en cours)
  • fonctionnalité ajout de jeux à louer

Backoffice :

  • validation d’inscription membre
  • système de bannissement si rating trop bas
  • gestion de la bdd (insert, update, delete)

Fonctionnalités supplémentaires:

  • appréciations loueur/emprunteur (rating)
  • fonction recherche et affichage profil autres membres
  • système de caution pour les jeux
  • système de communication / tchat / forum entre membres
